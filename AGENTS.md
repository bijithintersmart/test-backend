- Never duplicate logic.

---

# Redis

Database is always the source of truth.

For Create/Update/Delete

1. Update database

2. Invalidate cache

3. Reload latest record

4. Update Redis

5. Return latest object

Never return stale cache.

---

# Background Jobs

Never process inside HTTP requests

Use BullMQ workers for

- Email
- Notifications
- Reports
- AI
- Image processing
- PDF generation

---

# Database

Every financial or business-critical operation must use ACID transactions.

Protect against race conditions.

Use optimistic or pessimistic locking when appropriate.

---

# Security

Never trust client input.

Always validate.

Never log secrets.

Use bcrypt or argon2.

Implement rate limiting.

---

# Code Quality

Always

- Strong typing
- Async/await
- Dependency injection
- Repository pattern
- Service pattern
- Structured logging
- OpenAPI documentation

# Performance

Avoid

- N+1 queries
- Blocking operations
- Long transactions
- Huge responses

Prefer

- Pagination
- Indexes
- Streaming
- Caching
- Background jobs
